Mehmet Haluk Bakırcı | Information Systems & Technologies Student
This repository is a comprehensive technical log of my 21-day intensive journey into Sui Move smart contract development. From foundational syntax to advanced object-oriented blockchain architecture, this journey represents a transition from a learner to a capable Move developer.

🏗️ Project Milestones
📦 Week 1: Habit Tracker (The Fundamentals)
Focused on the core safety and efficiency of the Move language.

Core Concepts: Implementation of modules, primitive types, and vector-based data management.

Safety First: Integrated comprehensive Unit Testing to ensure logic integrity from day one.

Achievement: Built a decentralized system to track and verify daily habits on-chain.

💰 Week 2: Task Bounty Board (Ownership & Logic)
Deep dive into ownership models and complex state management.

Advanced Logic: Utilized Enum for state transitions and the Option<T> type for safe, crash-proof data retrieval.

Data Aggregation: Developed functions to calculate total rewards and track task completion metrics across the platform.

Security: Enforced strict visibility rules and ownership checks for task creators.

🚜 Week 3: Farm Simulator (Sui Object Model)
Architecture of a full-scale Sui application utilizing unique network features.

Object Architecture: Engineered structs with UID and key abilities to create unique, ownable on-chain assets.

Public API: Designed entry functions for seamless interaction with blockchain explorers and wallets.

Event-Driven: Implemented event::emit to broadcast real-time transaction data, facilitating off-chain indexing and monitoring.

🛠️ Technical Skill Set
Smart Contracts: Sui Move Framework, Ownership Models, Shared Objects.

Development Workflow: Sui CLI, Git, Test-Driven Development (TDD).

On-Chain Logic: Advanced Error Handling, Event Emission, Query Functions.
